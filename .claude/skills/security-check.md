# 安全审查清单

在涉及以下场景时，逐项检查：

## 1. 文件上传安全

- [ ] 文件类型白名单校验（扩展名 + MIME type）
- [ ] 文件大小限制（后端二次校验，不能只靠前端）
- [ ] 上传目录不放在 web 可访问路径下
- [ ] 文件名随机化，不使用用户原始文件名
- [ ] 图片文件用库重新编码（防图片马）

## 2. SQL 注入防护

- [ ] 所有查询使用参数化查询（SQLAlchemy ORM 自动安全）
- [ ] 原生 SQL 必须用参数绑定，绝不拼接字符串
- [ ] 动态排序字段白名单校验（`order_by` 不能直接拼接）

## 3. API 安全

- [ ] DeepSeek API Key 存在 `.env` 中，不硬编码
- [ ] `.env` 文件在 `.gitignore` 中
- [ ] 敏感操作需要确认（如删除、导出）
- [ ] CORS 配置限制允许的来源

## 4. 前端安全

- [ ] 用户输入用 Ant Design Form 校验规则
- [ ] 显示用户数据时注意 XSS（React 默认转义安全）
- [ ] 不在前端代码中硬编码 API Key

## 5. 导出安全

- [ ] 导出文件内容不包含未脱敏的用户数据
- [ ] 导出操作有日志记录
